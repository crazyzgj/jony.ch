Example for Configuring M-LAG Upgrade in Maintenance Mode (VXLAN M-LAG Dual-Homing Access Scenario)

Networking Requirements

On the network shown in Figure 1, SwitchA and SwitchB establish an M-LAG, and SwitchC and SwitchD establish an M-LAG. SwitchG to SwitchJ are dual-homed to the M-LAGs through Layer 2 sub-interfaces to connect to the VXLAN network. On the network side, ECMP is used to implement load balancing, and OSPF/OSPFv3/BGP/BGP4+ is used to implement routing. On the user side, Eth-Trunks work in LACP mode. To ensure that services deployed on devices are not affected during an upgrade, you can use M-LAG upgrade in maintenance mode to improve reliability during the upgrade.

  1. Configure SwitchA to work in maintenance mode and deliver configurations to switch traffic from Switch A to Switch B.
  2. Upgrade SwitchA. After the upgrade is completed successfully, restore traffic on SwitchA.
  3. Configure SwitchB to work in maintenance mode and deliver configurations to switch traffic from SwitchB to SwitchA.
  4. Upgrade SwitchB. After the upgrade is completed successfully, restore traffic on SwitchB.
Figure 1 Network diagram for configuring M-LAG upgrade in maintenance mode

Configuration Roadmap

The configuration roadmap is as follows:
  1. Configure OSPF/OSPFv3/BGP/BGP4+ on the network side to implement Layer 3 connectivity. Configure VXLAN tunnels.
  2. Configure M-LAG to implement active-active access of user-side devices.
  3. Upgrade the M-LAG consisting of SwitchA and SwitchB.
    • Install a license on SwitchA to use M-LAG upgrade in maintenance mode.

      M-LAG upgrade in maintenance mode is controlled by a license. By default, M-LAG upgrade in maintenance mode is disabled on a newly purchased device. To use M-LAG upgrade in maintenance mode, apply for and purchase the license from the device supplier.

    • Configure SwitchA to enter the maintenance mode and configure M-LAG upgrade in maintenance mode. Perform the following operations in sequence. Set the cost of OSPF routes to the maximum value and lower the priorities of BGP and BGP4+ routes to be advertised. Enable the device to forcibly set Eth-Trunk interfaces that are added to an M-LAG to the down state to implement traffic switchover.
    • Upgrade SwitchA. After SwitchA is upgraded successfully, perform the following operations in sequence to switch traffic back to the device. Disable the device from forcibly setting Eth-Trunk interfaces that are added to an M-LAG to the down state. Restore the cost values of OSPF routes and the priorities of BGP and BGP4+ routes to be advertised.
    • Repeat the preceding steps to upgrade SwitchB to complete the upgrade of the M-LAG.

Procedure

  1. Configure routing protocols and VXLAN tunnels.

    The routing protocol and VXLAN tunnel configurations are not provided here. For details, see Example for Configuring IPv4 VXLAN in Distributed Gateway Deployment Mode and Example for Configuring IPv6 VXLAN in Distributed Gateway Deployment Mode.

  2. Configure M-LAG.

    1. Create Eth-Trunk interfaces and add physical Ethernet interfaces to them.

      # Create Eth-Trunk interfaces in LACP mode on SwitchA and add member interfaces to the Eth-Trunk interfaces. The configuration of SwitchB is similar to that of SwitchA.

      [~SwitchA] interface eth-trunk 1
[*SwitchA-Eth-Trunk1] mode lacp-static
[*SwitchA-Eth-Trunk1] trunkport 10ge 1/0/4
[*SwitchA-Eth-Trunk1] trunkport 10ge 1/0/5
[*SwitchA-Eth-Trunk1] quit
[*SwitchA] interface eth-trunk 10
[*SwitchA-Eth-Trunk10] mode lacp-static
[*SwitchA-Eth-Trunk10] lacp mixed-rate link enable
[*SwitchA-Eth-Trunk10] trunkport 10ge 1/0/2
[*SwitchA-Eth-Trunk10] quit
[*SwitchA] interface eth-trunk 20
[*SwitchA-Eth-Trunk20] mode lacp-static
[*SwitchA-Eth-Trunk20] lacp mixed-rate link enable
[*SwitchA-Eth-Trunk20] trunkport 10ge 1/0/3
[*SwitchA-Eth-Trunk20] quit
[*SwitchA] commit
    2. Configure V-STP.

      # Configure SwitchA. The configuration of SwitchB is similar to that of SwitchA.

      [~SwitchA] stp mode rstp
[*SwitchA] stp v-stp enable
[*SwitchA] interface eth-trunk 10
[*SwitchA-Eth-Trunk10] stp edged-port enable
[*SwitchA-Eth-Trunk10] quit
[*SwitchA] interface eth-trunk 20
[*SwitchA-Eth-Trunk20] stp edged-port enable
[*SwitchA-Eth-Trunk20] quit
[*SwitchA] commit
    3. Configure a DFS group.

      # Configure SwitchA. The configuration of SwitchB is similar to that of SwitchA.

      [~SwitchA] dfs-group 1
[*SwitchA-dfs-group-1] source ip 10.3.3.3 peer 10.4.4.4
[*SwitchA-dfs-group-1] dual-active detection enhanced enable
[*SwitchA-dfs-group-1] quit
[*SwitchA] commit
    4. Configure a peer-link between SwitchA and SwitchB.

      # Configure SwitchA. The configuration of SwitchB is similar to that of SwitchA.

      [~SwitchA] interface eth-trunk 1
[~SwitchA-Eth-Trunk1] peer-link 1
[*SwitchA-Eth-Trunk1] quit
[*SwitchA] commit
    5. Bind user-side Eth-Trunk interfaces to the DFS group.

      # Configure SwitchA. The configuration of SwitchB is similar to that of SwitchA.

      [~SwitchA] interface eth-trunk 10
[~SwitchA-Eth-Trunk10] dfs-group 1 m-lag 1
[*SwitchA-Eth-Trunk10] quit
[*SwitchA] interface eth-trunk 20
[*SwitchA-Eth-Trunk20] dfs-group 1 m-lag 2
[*SwitchA-Eth-Trunk20] quit
[*SwitchA] commit

    Verify the configuration.

  3. Upgrade the M-LAG consisting of SwitchA and SwitchB.

    1. Install a license on SwitchA. For details about how to install a license, see the License Usage Guide.
    2. Configure SwitchA to enter the maintenance mode and configure M-LAG upgrade in maintenance mode. Perform the following operations in sequence to switch traffic to the backup device SwitchB.
      1. Configure SwitchA to enter the maintenance mode.

        # Configure SwitchA.

        [~SwitchA] maintenance
      2. Set the cost of OSPF or OSPFv3 routes to the maximum value or lower the priorities of BGP and BGP4+ routes to be advertised. (You can perform the operation based on the routing protocol used on the live network.)
        [*SwitchA-maintenance] ospf advertise max-cost
[*SwitchA-maintenance] ospfv3 advertise max-cost
[*SwitchA-maintenance] commit
[*SwitchA-maintenance] advertise bgp ipv4-family unicast lowest-priority enable
[*SwitchA-maintenance] advertise bgp ipv4-family vpnv4 lowest-priority enable
[*SwitchA-maintenance] advertise bgp ipv6-family unicast lowest-priority enable
[*SwitchA-maintenance] advertise bgp ipv6-family vpnv6 lowest-priority enable
[*SwitchA-maintenance] advertise bgp l2vpn-family evpn lowest-priority enable
[*SwitchA-maintenance] commit
      3. Enable the device to forcibly set Eth-Trunk interfaces that are added to an M-LAG to the down state to divert the uplink traffic from user-side devices to the standby link.
        [*SwitchA-maintenance] lacp force-down
[*SwitchA-maintenance] quit
[*SwitchA] commit
    3. Upgrade SwitchA, including loading the system software and restarting the device.

      For details about the precautions and upgrade operations, see the upgrade guide. To obtain the upgrade guide, log in to Huawei enterprise technical support website (https://support.huawei.com/enterprise), and obtain the upgrade guide based on the product model and version.

    4. After SwitchA is upgraded successfully (for details about how to verify the upgrade, see the upgrade guide), perform the following operations in sequence to restore traffic. Disable the device from forcibly setting Eth-Trunk interfaces that are added to an M-LAG to the down state. Restore the priorities of BGP or BGP4+ routes to be advertised and the cost values of OSPF or OSPFv3 routes.
      1. Configure SwitchA to enter the maintenance mode.
        [*SwitchA] maintenance
      2. Disable the device from forcibly setting Eth-Trunk interfaces that are added to an M-LAG to the down state.
        [*SwitchA-maintenance] undo lacp force-down
[*SwitchA-maintenance] commit

        Check whether the ARP, ND, and MAC address entries synchronized by the M-LAG member interface are restored. After the entries are restored, switch traffic back to the device.

      3. Restore the priorities of BGP or BGP4+ routes to be advertised and the cost values of OSPF or OSPFv3 routes.
        [*SwitchA-maintenance] undo advertise bgp l2vpn-family evpn lowest-priority enable
[*SwitchA-maintenance] undo advertise bgp ipv6-family vpnv6 lowest-priority enable
[*SwitchA-maintenance] undo advertise bgp ipv6-family unicast lowest-priority enable
[*SwitchA-maintenance] undo advertise bgp ipv4-family vpnv4 lowest-priority enable
[*SwitchA-maintenance] undo advertise bgp ipv4-family unicast lowest-priority enable
[*SwitchA-maintenance] commit
[*SwitchA-maintenance] undo ospf advertise max-cost
[*SwitchA-maintenance] undo ospfv3 advertise max-cost
[*SwitchA-maintenance] quit
[*SwitchA] commit
    5. Repeat the preceding steps to upgrade SwitchB to complete the upgrade of the M-LAG.
