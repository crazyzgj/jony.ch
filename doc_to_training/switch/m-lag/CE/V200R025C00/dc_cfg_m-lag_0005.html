Configuration Notes

Root bridge mode

When the root bridge mode is used to configure M-LAG, the M-LAG master and backup devices must be used as root bridges and configured with the same bridge ID on the STP network so that the two devices are simulated into one root bridge. The M-LAG master and backup devices are not affected by the network topology change on the Layer 2 network.
  1. Configuring the Root Bridge and Bridge ID
  2. Configuring a DFS Group
  3. Configuring M-LAG Consistency Check
  4. Configuring an Interface as a Peer-link Interface
  5. Configuring an M-LAG Member Interface
  6. (Optional) Configuring Dual-Active Gateways
  7. (Optional) Configuring a Dynamic Routing Protocol for Communication with an M-LAG
  8. (Optional) Configuring ARP and ND Entry Synchronization
  9. (Optional) Configuring the Interface Status When the Peer-Link Fails
  10. (Optional) Enabling Enhanced M-LAG Layer 3 Forwarding
  11. (Optional) Enabling the MAC FRR Function
  12. (Optional) Configuring M-LAG Uplink Interfaces
  • Configure the M-LAG master and backup devices as root bridges and configure the same bridge ID for them.
  • STP must be disabled on peer-link interfaces.

V-STP mode (recommended)

V-STP virtualizes the M-LAG master and backup devices enabled with STP into one device to perform STP calculation.

  2. Configuring a DFS Group
  3. Configuring M-LAG Consistency Check

  5. Configuring an M-LAG Member Interface

  8. (Optional) Configuring ARP and ND Entry Synchronization
  9. (Optional) Configuring the Interface Status When the Peer-Link Fails
  10. (Optional) Enabling Enhanced M-LAG Layer 3 Forwarding
  11. (Optional) Enabling the MAC FRR Function
  12. (Optional) Configuring M-LAG Uplink Interfaces

You must enable STP on peer-link interfaces and enable V-STP.

V-VBST mode

VBST virtualizes the M-LAG master and backup devices enabled with VBST into one device to perform VBST calculation.
  1. Configuring V-VBST
  2. Configuring a DFS Group
  3. Configuring M-LAG Consistency Check
  4. Configuring the Peer-Link
  5. Configuring an M-LAG Member Interface
  6. (Optional) Configuring Dual-Active Gateways
  7. (Optional) Configuring a Dynamic Routing Protocol for Communication with an M-LAG
  8. (Optional) Configuring ARP/ND Entry Synchronization
  9. (Optional) Configuring the Interface Status in a Peer-Link Failure Scenario
  10. (Optional) Enabling Enhanced M-LAG Layer 3 Forwarding
  11. (Optional) Enabling the MAC FRR Function
  12. (Optional) Configuring M-LAG Uplink Interfaces

Set the STP working mode of the switches to VBST.
  • In root bridge mode, the two devices in an M-LAG must function as root bridges on a Layer 2 network, and multi-level M-LAG in all-root bridge mode is not supported.
  • In V-STP mode, the two devices in an M-LAG do not need to function as root bridges on a Layer 2 network. The networking is flexible and multi-level M-LAG is supported. V-STP is recommended because it can eliminate loops caused by incorrect M-LAG configurations or connections.
  • V-VBST can be used only in M-LAG networking to meet the requirements of multi-level M-LAG interconnection scenarios and scenarios where devices in an M-LAG function as non-root bridges. In addition, V-VBST can solve the port blocking problem after STP is enabled.
