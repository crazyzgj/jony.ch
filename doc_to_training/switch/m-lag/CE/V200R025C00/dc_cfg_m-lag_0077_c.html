(Optional) Configuring a Dynamic Routing Protocol for Communication with an M-LAG

Prerequisites

Before configuring OSPF, OSPFv3, BGP, or BGP4+ over M-LAG, complete the following tasks:

  • Establish an M-LAG and an OSPF, OSPFv3, BGP, or BGP4+ network.
  • Enable OSPF, OSPFv3, BGP, or BGP4+ on the user-side device.

Context

On the network shown in Figure 1, SwitchA (a user-side device) is dual-homed to an M-LAG and has a small number of static routes configured so that it can communicate with the M-LAG through Layer 3 routes. However, the network using static routes is difficult to configure and maintain and is lack of flexible and fast deployment capabilities, thereby cannot meet the requirements of rapidly growing services. To address this problem, M-LAG member devices need to establish neighbor relationships of dynamic routing protocols with SwitchA. Therefore, M-LAG member interfaces need to support dynamic routing protocols.

Figure 1 Configuring a dynamic routing protocol for communication with an M-LAG

In the scenario where a device is dual-homed to an OSPF, OSPFv3, BGP, or BGP4+ network through an M-LAG, you must configure IP addresses for VLANIF interfaces corresponding to M-LAG member interfaces on M-LAG master and backup devices. When configuring OSPF, OSPFv3, BGP, or BGP4+, configure the same IP address for the VLANIF interfaces so that M-LAG member devices can establish an OSPF or OSPFv3 neighbor relationship with each other and establish OSPF or OSPFv3 neighbor relationships or BGP or BGP4+ peer relationships with SwitchA.

Configuring a dynamic routing protocol for communication with an M-LAG is not supported by the CE9860EI, CE5855E, and CE5855F.

Configuring a dynamic routing protocol for communication with an M-LAG is supported only in the VLANIF interface view on the CE8850EI.

Topology isolation is required between the network side and the M-LAG side. You can perform one of the following configurations to implement topology isolation:

  • Deploy different dynamic routing protocols on the network side and the M-LAG side.
  • Deploy OSPF/OSPFv3 on the network side and the M-LAG side, but configure different OSPF/OSPFv3 processes on the two sides.
  • Deploy OSPF/OSPFv3 on the network side and the M-LAG side and configure the same OSPF/OSPFv3 process on the two sides, but configure different OSPF/OSPFv3 areas on the two sides.

You are advised to configure BFD for M-LAG between M-LAG member devices and SwitchA to quickly detect link faults.

If a dynamic routing protocol has been configured for communication with an M-LAG on the CE6870EI, BFD cannot be enabled for a peer or peer group.

If a VPN instance is specified when OSPF/OSPFv3 is configured, you are advised to run the vpn-instance-capability simple command in the OSPF/OSPFv3 view to enable the device to directly calculate routes without conducting routing loop detection.

Procedure

  • Configure OSPF over M-LAG.
    1. Run system-view

      The system view is displayed.

    2. Run interface vlanif vlan-id

      The VLANIF interface view is displayed.

    3. Run ip address ip-address { mask | mask-length } [ tag tag-value ]

      An IP address is configured.

    4. Run mac-address mac-address

      A virtual MAC address is configured for the VLANIF interface.

    5. Run ospf source sub-address ipv4-address-sub

      A secondary IP address of the M-LAG device is specified as the source IP address.

    6. Run m-lag ip address ip-address { mask-value | mask-v4 }

      The source IP address is configured as the M-LAG IP address.

      • An M-LAG IPv4 address is a secondary IP address. An interface can be configured with only one M-LAG secondary IPv4 address and a maximum of 255 secondary IP addresses.
      • An M-LAG secondary IPv4 address must be unique on the entire network, and cannot be the same as any secondary address on the local device or other devices on the network.
      • A multicast, broadcast, or loopback address cannot be configured as an M-LAG IPv4 address.
      • If an M-LAG secondary IPv4 address has been configured on an interface and a new M-LAG secondary IPv4 address is configured on the interface, the new address overwrites the original one.
      • Before running this command, you must configure a primary IP address, and the IP address specified in this command must be on the same network segment as the primary IP address.
      • When m-lag ip address has been configured under the VBDIF interface, if you unbind a VNI from a BD, the switch automatically deletes the M-LAG ipv4 address under the VBDIF interface.
      • The m-lag ip address command can be configured on a VBDIF interface only after a VNI is bound to the corresponding BD.
      • If the ip address ignore primary-sub enable command is configured on the device, the m-lag ip address command cannot be configured.
      • If an M-LAG IPv4 address configured for an interface using the m-lag ip address command is the same as a primary or secondary IP address on the interface, the M-LAG IPv4 address cannot be delivered.

    7. Run quit

      Exit the interface view.

    8. Run commit

      The configuration is committed.

    9. Run ospf [ process-id | router-id router-id | vpn-instance vpn-instance-name ]

      An OSPF process is created and the OSPF view is displayed.

    10. Run area area-id

      The OSPF area view is displayed.

    11. Run network ip-address wildcard-mask

      A network segment contained in the area is configured.

    12. Run commit

      The configuration is committed.

  • Configure OSPFv3 over M-LAG.
    1. Run system-view

      The system view is displayed.

    2. Run ospfv3 [ process-id ] [ vpn-instance vpn-instance-name ]

      An OSPFv3 process is created and the OSPFv3 view is displayed.

    3. Run router-id router-id

      A router ID is configured.

    4. Run area area-id

      The OSPFv3 area view is displayed.

    5. Run commit

      The configuration is committed.

    6. Run interface vlanif vlan-id

      The VLANIF interface view is displayed.

    7. Run ipv6 enable

      IPv6 is enabled on the interface.

    8. Run ipv6 address { ipv6-address prefix-length | ipv6-address/prefix-length } [ tag tag-value ]

      An IPv6 address is configured.

    9. Run mac-address mac-address

      A virtual MAC address is configured for the VLANIF interface.

    10. Run ospfv3 process-id area area-id [ instance instance-id ]

      OSPFv3 is enabled on the interface.

    11. Run m-lag ipv6 address ipv6-address link-local

      The source IPv6 address is configured as the M-LAG IPv6 address.

      • Only one M-LAG link-local address can be configured on each interface.
      • The M-LAG link-local address cannot be the same as any link-local address on the local or remote M-LAG member device.
      • Before configuring an M-LAG link-local address for an interface, you must enable IPv6 on the interface.
      • When m-lag ipv6 address link-local has been configured under the VBDIF interface, if you unbind a VNI from a BD, the switch automatically deletes the ipv6 link-local address under the VBDIF interface.
      • The m-lag ipv6 address link-local command can be configured on a VBDIF interface only after a VNI is bound to the corresponding BD.
      • If an M-LAG IPv6 link-local address configured for an interface using the m-lag ipv6 address link-local command is the same as a primary or secondary IPv6 address on the interface, the M-LAG IPv6 link-local address cannot be delivered.
      • If the ipv6 address link-local command is configured on an interface, the m-lag ipv6 address link-local command cannot be configured on the interface.
      • When an IPv6 address is configured using the m-lag ipv6 address link-local command, the prefix of the specified IPv6 address must match FE80::/10. When an IPv6 address is configured using the m-lag ipv6 address command, the prefix of the specified IPv6 address cannot be FE80::/10.
      • On a device, all VLANIF or VBDIF interfaces bound to the same VPN instance cannot be configured with the same M-LAG link-local address. When Layer 3 interfaces on the master and backup devices in an M-LAG are bound to the same VPN instance, the link-local address of the Layer 3 interface on the local M-LAG device cannot be the same as the M-LAG link-local address configured on the remote M-LAG device.

    12. Run commit

      The configuration is committed.

  • Configure BGP over M-LAG.
    1. Run system-view

      The system view is displayed.

    2. Run interface vlanif vlan-id

      The VLANIF interface view is displayed.

    3. Run ip address ip-address { mask | mask-length } [ tag tag-value ]

      A secondary IP address is configured.

    4. Run mac-address mac-address

      A virtual MAC address is configured for the VLANIF interface.

    5. Run m-lag ip address ip-address { mask-value | mask-v4 }

      The source IP address is configured as the M-LAG IP address.

      • An M-LAG IPv4 address is a secondary IP address. An interface can be configured with only one M-LAG secondary IPv4 address and a maximum of 255 secondary IP addresses.
      • An M-LAG secondary IPv4 address must be unique on the entire network, and cannot be the same as any secondary address on the local device or other devices on the network.
      • A multicast, broadcast, or loopback address cannot be configured as an M-LAG IPv4 address.
      • If an M-LAG secondary IPv4 address has been configured on an interface and a new M-LAG secondary IPv4 address is configured on the interface, the new address overwrites the original one.
      • Before running this command, you must configure a primary IP address, and the IP address specified in this command must be on the same network segment as the primary IP address.
      • When m-lag ip address has been configured under the VBDIF interface, if you unbind a VNI from a BD, the switch automatically deletes the M-LAG ipv4 address under the VBDIF interface.
      • The m-lag ip address command can be configured on a VBDIF interface only after a VNI is bound to the corresponding BD.
      • If the ip address ignore primary-sub enable command is configured on the device, the m-lag ip address command cannot be configured.
      • If an M-LAG IPv4 address configured for an interface using the m-lag ip address command is the same as a primary or secondary IP address on the interface, the M-LAG IPv4 address cannot be delivered.

    6. Run quit

      Exit the interface view.

    7. Run commit

      The configuration is committed.

    8. Run bgp { as-number-plain | as-number-dot }

      BGP is enabled, the local AS number is specified, and the BGP view is displayed.

    9. Run router-id ipv4-address

      A BGP router ID is configured.

    10. Run peer ipv4-address as-number as-number

      A BGP peer is created.

    11. Run peer ipv4-address connect-interface { interface-type interface-number [ ipv4-source-address ] | ipv4-source-address }

      The source interface and source IP address used to set up a TCP connection between BGP peers are specified.

    12. Run ipv4-family unicast

      The BGP-IPv4 unicast address family view is displayed.

    13. Run peer ipv4-address enable

      MP-BGP is enabled for the BGP peer so that it becomes an MP-BGP peer.

    14. Run commit

      The configuration is committed.

  • Configure BGP4+ over M-LAG.
    1. Run system-view

      The system view is displayed.

    2. Run interface vlanif vlan-id

      The VLANIF interface view is displayed.

    3. Run ipv6 enable

      IPv6 is enabled on the interface.

    4. Run ipv6 address { ipv6-address prefix-length | ipv6-address/prefix-length } [ tag tag-value ]

      An IPv6 address is configured.

    5. Run mac-address mac-address

      A virtual MAC address is configured for the VLANIF interface.

    6. Run m-lag ipv6 address ipv6-address [ link-local ]

      The source IPv6 address is configured as the M-LAG IPv6 address.

      • An M-LAG IPv6 address can only be a global unicast address. An interface can be configured with only one M-LAG IPv6 address and a maximum of 16 global unicast addresses.
      • An M-LAG IPv6 address must be unique on the entire network, and cannot be the same as any IPv6 address on the local device or other devices on the network.
      • A multicast, anycast, or loopback address cannot be configured as an M-LAG IPv6 address.
      • If an M-LAG IPv6 address has been configured on an interface and a new M-LAG IPv6 address is configured on the interface, the new address overwrites the original one.
      • A link-local address cannot be configured as an M-LAG IPv6 address.
      • Before configuring an M-LAG IPv6 address for an interface, you must enable IPv6 on the interface.
      • When m-lag ipv6 address has been configured under the VBDIF interface, if you unbind a VNI from a BD, the switch automatically deletes the M-LAG ipv6 address under the VBDIF interface.
      • The m-lag ipv6 address command can be configured on a VBDIF interface only after a VNI is bound to the corresponding BD.
      • If an M-LAG IPv6 address configured for an interface using the m-lag ipv6 address command is the same as a primary or secondary IPv6 address on the interface, the M-LAG IPv6 address cannot be delivered.
      • When an IPv6 address is configured using the m-lag ipv6 address command, the prefix of the specified IPv6 address cannot be FE80::/10. When an IPv6 address is configured using the m-lag ipv6 address link-local command, the prefix of the specified IPv6 address must match FE80::/10.
      • Only one M-LAG link-local address can be configured on each interface.
      • The M-LAG link-local address cannot be the same as any link-local address on the local or remote M-LAG member device.
      • Before configuring an M-LAG link-local address for an interface, you must enable IPv6 on the interface.
      • When m-lag ipv6 address link-local has been configured under the VBDIF interface, if you unbind a VNI from a BD, the switch automatically deletes the ipv6 link-local address under the VBDIF interface.
      • The m-lag ipv6 address link-local command can be configured on a VBDIF interface only after a VNI is bound to the corresponding BD.
      • If an M-LAG IPv6 link-local address configured for an interface using the m-lag ipv6 address link-local command is the same as a primary or secondary IPv6 address on the interface, the M-LAG IPv6 link-local address cannot be delivered.
      • If the ipv6 address link-local command is configured on an interface, the m-lag ipv6 address link-local command cannot be configured on the interface.
      • When an IPv6 address is configured using the m-lag ipv6 address link-local command, the prefix of the specified IPv6 address must match FE80::/10. When an IPv6 address is configured using the m-lag ipv6 address command, the prefix of the specified IPv6 address cannot be FE80::/10.
      • On a device, all VLANIF or VBDIF interfaces bound to the same VPN instance cannot be configured with the same M-LAG link-local address. When Layer 3 interfaces on the master and backup devices in an M-LAG are bound to the same VPN instance, the link-local address of the Layer 3 interface on the local M-LAG device cannot be the same as the M-LAG link-local address configured on the remote M-LAG device.

    7. Run quit

      Exit the interface view.

    8. Run commit

      The configuration is committed.

    9. Run bgp { as-number-plain | as-number-dot }

      BGP is enabled, the local AS number is specified, and the BGP view is displayed.

    10. Run router-id ipv4-address

      A BGP router ID is configured.

    11. Run peer ipv6-address as-number as-number

      A BGP peer is created.

    12. Run peer ipv6-address connect-interface { interface-type interface-number [ ipv6-source-address ] | ipv6-source-address }

      The source interface and source IP address used to set up a TCP connection between BGP peers are specified.

    13. Run ipv6-family unicast

      The BGP-IPv6 unicast address family view is displayed.

    14. Run peer ipv6-address enable

      MP-BGP is enabled for the BGP peer so that it becomes an MP-BGP peer.

    15. Run commit

      The configuration is committed.
