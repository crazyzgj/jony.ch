(Optional) Configuring M-LAG Uplink Interfaces

Context

When all uplink interfaces on the DFS master device in an M-LAG fail, if the peer-link interface fails, the DAD heartbeat status is normal, and the cards where peer-link interfaces on the M-LAG master and backup devices reside have the same fault status, two master devices exist in the M-LAG, and interfaces on the M-LAG backup device enter the Error-Down state. As a result, traffic cannot be forwarded.

As shown in the following figure, if all uplink interfaces on the master device are Down but uplink interfaces on the backup device are Up, traffic can be forwarded through the backup device. If the peer-link fails and the DAD link is normal, interfaces on the backup device enter the Error-Down state. As a result, traffic cannot be forwarded. You can configure an uplink interface as an M-LAG uplink interface. The state of the uplink interface is used to determine whether two master devices exist. If the peer-link fails, the states of M-LAG uplink interfaces are preferentially compared. If not all uplink interfaces on the local device fail or all uplink interfaces on the remote device fail, the local device wins the master/backup election. Interfaces on the device that fails to win the election are set to Error-Down state to ensure normal traffic forwarding and improve reliability in failure scenarios.

Figure 1 Networking diagram when an M-LAG uplink fails and the peer-link fails

Procedure

  1. Run system-view

    The system view is displayed.

  2. Run interface interface-type interface-number

    The interface view is displayed.

  3. Run uplink-port enable

    The interface is configured as an M-LAG uplink interface.

    By default, an interface is not configured as an M-LAG uplink interface.

    • A maximum of 256 uplink interfaces can be configured in an M-LAG.
    • Physical stack member interfaces, M-LAG peer-link interfaces, Eth-Trunk member interfaces, management interfaces, and logical interfaces cannot be configured as uplink interfaces.
    • If no uplink interface is configured on a member switch, uplink interfaces on the switch are considered Up.
    • Uplink interfaces transmit uplink traffic. Do not configure the interfaces that do not transmit uplink traffic as uplink interfaces.

  4. Run commit

    The configuration is committed.
