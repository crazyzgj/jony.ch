Verifying the Configuration of M-LAG Configured Through V-VBST

Procedure

  • Run the display dfs-group dfs-group-id [ node node-id m-lag [ brief ] | peer-link ] command to check M-LAG information.
  • Run the display stp vlan vlan-id v-vbst command to check the V-VBST calculation result of the spanning tree in a specified VLAN.

Follow-up Procedure

After the M-LAG configuration is complete, if the peer-link fails but the heartbeat status is normal, some interfaces on the M-LAG backup device are in Error-Down state. When detecting a fault on an interface, the device sets the interface to the Error-Down state. The interface cannot send or receive packets, and the interface indicator is off. You can run the display error-down recovery command to view information about all interfaces in Error-Down state on the device.

When M-LAG is used for dual-homing access on a common Ethernet or IP network, if the peer-link fails but the DAD heartbeat status is normal, interfaces excluding the management interface, peer-link interface, and stack interface on the M-LAG backup device enter the Error-Down state. When the faulty peer-link recovers, the M-LAG member interface in the Error-Down state automatically restores to the Up state after 240s by default, and the other interfaces in the Error-Down state automatically restore to the Up state immediately.

If an interface enters the Error-Down state, you are advised to locate the cause of the peer-link fault first. You are not advised to manually restore the interface to the Up state or run the error-down auto-recovery cause m-lag interval interval-value command in the system view to enable the interface to automatically go Up. Otherwise, faults such as excess service packets, packet loss, and service interruption may occur. Exercise caution when performing this operation.
