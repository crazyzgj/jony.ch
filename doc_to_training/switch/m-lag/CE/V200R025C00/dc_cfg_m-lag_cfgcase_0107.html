Example for Configuring BGP over M-LAG in V-VBST Mode

Networking Requirements

On the network shown in Figure 1, SwitchA, SwitchB, and SwitchC establish an M-LAG. The M-LAG member interfaces on SwitchB and SwitchC support dynamic routing protocols. A dynamic routing protocol is configured on SwitchA so that it can communicate with the M-LAG through Layer 3 routes.

Figure 1 Network diagram for configuring a dynamic routing protocol for communication with an M-LAG

Configuration Roadmap

The configuration roadmap is as follows:
  1. Configure a routing protocol on SwitchB, SwitchC, and SwitchD to implement Layer 3 connectivity.
  2. Create Eth-Trunk interfaces.

  3. Configure V-VBST.

  4. Configure M-LAG.
    • Configure a DFS group on SwitchB and SwitchC.
    • Configure the link between SwitchB and SwitchC as the peer-link.
    • Bind the user-side Eth-Trunk interface to the DFS group on SwitchB and SwitchC.
  5. Configure an IP address for BGP over M-LAG on SwitchB and SwitchC. Configure different M-LAG IP addresses on SwitchB and SwitchC. Otherwise, peer relationships of the routing protocol cannot be established.
  6. Configure SwitchA to establish BGP peer relationships with M-LAG member devices (SwitchB and SwitchC).

Procedure

  1. Configure a routing protocol to implement Layer 3 connectivity on the network.

    When BGP is configured to import direct routes, a routing policy needs to be added to prevent the M-LAG IP address from being advertised.

    # Configure a BGP peer for SwitchB.

    <HUAWEI> system-view
[~HUAWEI] sysname SwitchB
[*HUAWEI] commit
[~SwitchB] bgp 100  
[*SwitchB-bgp] router-id 11.12.12.12 
[*SwitchB-bgp] peer 192.168.1.2 as-number 200 
[*SwitchB-bgp] quit 
[*SwitchB] commit

    # Configure a BGP peer for SwitchC.

    <HUAWEI> system-view
[~HUAWEI] sysname SwitchC
[*HUAWEI] commit
[~SwitchC] bgp 100  
[*SwitchC-bgp] router-id 11.14.14.14 
[*SwitchC-bgp] peer 192.168.2.2 as-number 200 
[*SwitchC-bgp] quit 
[*SwitchC] commit

    # Configure BGP peers for SwitchD.

    <HUAWEI> system-view
[~HUAWEI] sysname SwitchD
[*HUAWEI] commit
[~SwitchD] bgp 200  
[*SwitchD-bgp] router-id 11.13.13.13 
[*SwitchD-bgp] peer 192.168.1.1 as-number 100 
[*SwitchD-bgp] peer 192.168.2.1 as-number 100 
[*SwitchD-bgp] quit 
[*SwitchD] commit

    After BGP is configured successfully, SwitchB, SwitchC, and SwitchD can establish BGP peer relationships and the peer status is Established.

  2. Create Eth-Trunk interfaces and add physical Ethernet interfaces to the Eth-Trunk interfaces.

    The uplink interfaces that connect servers to SwitchB and SwitchC must be added to an Eth-Trunk interface, and the working mode of the Eth-Trunk interface must be the same as that of the Eth-Trunk interfaces on both switches.

    # Create Eth-Trunk interfaces in LACP mode on SwitchB and add member interfaces to the Eth-Trunk interfaces. The configuration of SwitchC is similar to that of SwitchB. The lacp mixed-rate link enable command needs to be configured only when interfaces of different rates are added to an Eth-Trunk interface in LACP mode.

    [~SwitchB] interface eth-trunk 1
[*SwitchB-Eth-Trunk1] mode lacp-static
[*SwitchB-Eth-Trunk1] trunkport 10ge 1/0/3
[*SwitchB-Eth-Trunk1] trunkport 10ge 1/0/4
[*SwitchB-Eth-Trunk1] quit
[*SwitchB] interface eth-trunk 10
[*SwitchB-Eth-Trunk10] mode lacp-static
[*SwitchB-Eth-Trunk10] lacp mixed-rate link enable
[*SwitchB-Eth-Trunk10] trunkport 10ge 1/0/2
[*SwitchB-Eth-Trunk10] quit
[*SwitchB] commit

  3. Set the STP working mode of the switches to VBST.

    # Configure SwitchB.

    [~SwitchB] stp mode vbst
[*SwitchB] commit

    # Configure SwitchC.

    [~SwitchC] stp mode vbst
[*SwitchC] commit

  4. Configure a DFS group on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] dfs-group 1
[*SwitchB-dfs-group-1] source ip 10.200.1.1 peer 10.200.2.1
[*SwitchB-dfs-group-1] dual-active detection enhanced enable
[*SwitchB-dfs-group-1] quit
[*SwitchB] commit

  5. Configure the link between SwitchB and SwitchC as the peer-link.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] interface eth-trunk 1
[~SwitchB-Eth-Trunk1] peer-link 1
[*SwitchB-Eth-Trunk1] quit
[*SwitchB] commit

  6. Bind the user-side Eth-Trunk interface to the DFS group on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] interface eth-trunk 10
[~SwitchB-Eth-Trunk10] dfs-group 1 m-lag 1
[*SwitchB-Eth-Trunk10] quit
[*SwitchB] commit

  7. Configure an IP address for BGP over M-LAG on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] vlan 100
[*SwitchB-vlan100] quit
[*SwitchB] interface vlanif 102
[*SwitchB-Vlanif100] ip address 100.102.0.1 255.255.255.0
[*SwitchB-Vlanif100] m-lag ip address 100.102.0.3 255.255.255.0
[*SwitchB-Vlanif100] mac-address 0000-5e00-0101
[*SwitchB-Vlanif100] quit
[*SwitchB] commit
[~SwitchB] interface eth-trunk 10 
[*SwitchB-Eth-Trunk10] port trunk allow-pass vlan 102 
[*SwitchB-Eth-Trunk10] quit 
[*SwitchB] commit

  8. Configure SwitchA to establish BGP peer relationships with M-LAG member devices (SwitchB and SwitchC).

    # Configure SwitchA.
    [~SwitchA] bgp 100  
[*SwitchA-bgp] router-id 11.15.15.15 
[*SwitchA-bgp] peer 10.102.0.3 as-number 100  
[*SwitchA-bgp] peer 10.102.0.4 as-number 100

    # Configure SwitchB.

    [~SwitchB] bgp 100 
[~SwitchB-bgp] peer 10.102.0.2 as-number 100  
[*SwitchB-bgp] peer 10.102.0.2 connect-interface Vlanif102 10.102.0.3

    # Configure SwitchC.

    [~SwitchC] bgp 100 
[~SwitchC-bgp] peer 10.102.0.2 as-number 100  
[*SwitchC-bgp] peer 10.102.0.2 connect-interface Vlanif102 10.102.0.4

Configuration Files

  • SwitchB configuration file
    #
sysname Leaf
#
dfs-group 1
 source ip 10.200.1.1 peer 10.200.2.1
 dual-active detection enhanced enable
#
vlan batch 102
#
stp mode vbst
#
interface Vlanif102
 ip address 100.102.0.1 255.255.255.0
 mac-address 0000-5e00-0101
 m-lag ip address 100.102.0.3 255.255.255.0
#
interface Eth-Trunk1
 mode lacp-static
 peer-link 1
#
interface Eth-Trunk10
 port link-type trunk
 undo port trunk allow-pass vlan 1
 port trunk allow-pass vlan 102
 mode lacp-static
 lacp mixed-rate link enable
 dfs-group 1 m-lag 1
#
interface 10GE1/0/1
 undo portswitch
 ip address 192.168.1.1 255.255.255.0
#
interface 10GE1/0/2
 eth-trunk 10
#
interface 10GE1/0/3
 eth-trunk 1
#
interface 10GE1/0/4
 eth-trunk 1
#
bgp 100 
 router-id 11.12.12.12 
 peer 10.102.0.2 as-number 100 
 peer 10.102.0.2 connect-interface Vlanif102 10.102.0.3                                                      
 peer 192.168.1.2 as-number 200    
 #  
 ipv4-family unicast   
  peer 10.102.0.2 enable                                                                                
  peer 192.168.1.2 enable 
#
return
