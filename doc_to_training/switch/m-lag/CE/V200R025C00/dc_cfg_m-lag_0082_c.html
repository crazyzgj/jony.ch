(Optional) Enabling the MAC FRR Function

Context

To speed up convergence when an M-LAG member interface fails, you can enable the MAC FRR function so that backup FRR resources are requested for all MAC address entries with M-LAG member interfaces as outbound interfaces. Active and standby paths are established for traffic forwarding on downlink outbound interfaces. If an M-LAG member interface fails, the outbound interface can be quickly changed to a peer-link interface.

Only the CE6881, CE5881, CE6881K, CE6820, CE6863K, CE6881E, CE6863E, and CE6863 support the MAC FRR function.

Procedure

  1. Run system-view

    The system view is displayed.

  2. Run dfs-group dfs-group-id

    The DFS group view is displayed.

  3. Run m-lag mac-address frr

    The MAC FRR function is enabled.

    By default, the MAC FRR function is disabled.

    After the MAC FRR function is enabled in an M-LAG, backup FRR resources are requested for all MAC address entries with M-LAG member interfaces as outbound interfaces. The outbound interfaces can be changed to peer-link interfaces to establish active and standby paths for traffic forwarding. If the FEI side detects that an M-LAG member interface fails, dual-homing networking is changed to single-homing networking. The next hop in the corresponding MAC address entry is changed from the M-LAG member interface to the peer-link interface. This improves the switchover performance when faults occur.

    After the MAC FRR function is enabled using the m-lag mac-address frr command, run the reset mac-address command to clear MAC addresses. Otherwise, the MAC FRR function cannot work properly.

    After this function is enabled, backup FRR resources are requested only for MAC address entries with M-LAG member interfaces as outbound interfaces, but not for MAC address entries with sub-interfaces of M-LAG member interfaces as outbound interfaces.

  4. Run commit

    The configuration is committed.
