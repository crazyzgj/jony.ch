(Optional) Configuring Dual-Active Gateways

Prerequisites

The M-LAG member interface has been added to the corresponding VLAN, or the Layer 2 sub-interface of the Eth-Trunk to which the M-LAG member interface belongs has been added to the corresponding BD.

Context

On a dual-homing IP or VXLAN network, both the M-LAG master and backup devices need to function as Layer 3 gateways. In this case, VLANIF/VBDIF interfaces corresponding to M-LAG member interfaces must have the same IP address and MAC address. You can configure the same IP address for the VLANIF/VBDIF interfaces and run the mac-address command to configure the same virtual MAC address for them.

Procedure

  • Configure IP and MAC addresses for VLANIF/VBDIF interfaces to implement the dual-active gateway function.
    1. Run system-view

      The system view is displayed.

    2. Run interface { vlanif vlan-id | vbdif bd-id }

      The VLANIF or VBDIF interface view is displayed.

    3. Configure an IP address for the interface:

      • On IPv4 networks, run ip address ip-address { mask | mask-length } [ sub ] [ tag tag-value ]

        An IPv4 address is configured for the VLANIF/VBDIF interface.

      • On IPv6 networks, perform the following operations:

        1. Run ipv6 enable

          IPv6 is enabled for the VLANIF/VBDIF interface.

        2. Run ipv6 address { ipv6-address prefix-length | ipv6-address/prefix-length } [ tag tag-value ]

          Or run ipv6 address { ipv6-address prefix-length | ipv6-address/prefix-length } eui-64 [ tag tag-value ]

          A global unicast address is configured for the VLANIF/VBDIF interface.

      By default, no IP address is configured for an interface.

      VLANIF/VBDIF interfaces corresponding to M-LAG member interfaces of M-LAG master and backup devices must be configured with the same IP address.

    4. Run mac-address mac-address (VLANIF interface view)

      Or run mac-address mac-address (VBDIF interface view)

      A virtual MAC address is configured for the VLANIF/VBDIF interface.

      By default, the MAC address of a VLANIF/VBDIF interface is the same as the system MAC address.

      VLANIF/VBDIF interfaces corresponding to M-LAG member interfaces of M-LAG master and backup devices must be configured with the same virtual MAC address.

      When devices in an M-LAG are connected to third-party firewalls enabled with the sticky load balancing function and network attached storage (NAS) devices or load balancers enabled with the Auto Last Hop function, these devices do not learn the gateway MAC address based on the normal ARP request logic. Instead, they use the source MAC address of the first packet from the gateway as the gateway MAC address. Therefore, if an M-LAG device fails or restarts when the two M-LAG devices use their own system MAC addresses as the source MAC addresses to encapsulate packets, services are interrupted for several minutes. You are advised to disable the sticky load balancing function of the firewalls and the Auto Last Hop function of the NAS devices or load balancers in the preceding scenario.
      • For the CE6870EI, the any unicast MAC address (non-virtual MAC address) cannot be configured on an interface. If the functions cannot be disabled, configure the same any unicast MAC address (non-virtual MAC address) on the VLANIF/VBDIF gateway interfaces connecting the two M-LAG devices to the firewalls, NAS devices, or load balancers.
      • For other models, if the functions cannot be disabled, you are advised to configure M-LAG devices as dual-active gateways using the same IP address and the same virtual MAC address.

    5. (Optional) Run protocol up-delay-time time-value

      The delay before the Layer 3 protocol status of an interface can go Up is set.

      By default, the delay before the Layer 3 protocol status of a VLANIF interface can go Up is 1s. When there are many ARP entries, the delay is increased.

      When the faulty device or peer-link recovers, many ARP entries need to be synchronized in a batch. You can configure the delay so that the protocol status of the interface goes Up after ARP entries are synchronized. This prevents protocol packets from being discarded, reduces the packet loss during link recovery, and improves convergence performance.

      After a device restarts or a card resets, the physical status of an interface changes to Up, but the status of the upper-layer protocol module does not meet forwarding requirements, resulting in packet loss. To ensure switchback performance, the default delay for an M-LAG member interface to report the Up state is 240s. If a delay after which the Layer 3 protocol status changes to Up is configured on a VLANIF interface, ensure that the delay for an M-LAG member interface to report the Up event is longer than the delay configured on the VLANIF interface. Otherwise, triggering of learning for ND entries that fail to be synchronized depends on ND Miss messages.

    6. Run quit

      Return to the system view.

    7. (Optional) Run ipv6 nd packet strict-match mode enable

      The strict match mode for IPv6 ND packets is enabled.

      When IPv6 dual-active gateways are configured in an M-LAG, you can enable the strict match mode for IPv6 ND packets to more accurately match NA packets and reduce the number of NA packets that are incorrectly sent to the CPU.

      By default, the strict match mode for IPv6 ND packets is disabled.

      Only the CE6870EI supports this command.

    8. Run dfs-group dfs-group-id

      The DFS group view is displayed.

    9. (Optional) Run vbdif mac-address advertise disable

      The M-LAG device is disabled from advertising the MAC address of a VBDIF interface synchronized from the peer device to remote VXLAN-enabled devices.

      By default, an M-LAG device advertises the MAC address of a VBDIF interface to remote VXLAN-enabled devices through EVPN after receiving the MAC address from the peer device.

    10. (Optional) Run vbdif mac-address synchronization disable

      The M-LAG device is configured not to synchronize the system MAC address of a VBDIF interface to the peer M-LAG device.

      By default, when a VBDIF interface is configured on an M-LAG device, the M-LAG device synchronizes the system MAC address of the VBDIF interface to the peer M-LAG device through the M-LAG synchronization channel.

    11. (Optional) Run vlanif mac-address synchronization disable

      The M-LAG device is configured not to synchronize the system MAC address of a VLANIF interface to the peer M-LAG device.

      By default, when a VLANIF interface is configured on an M-LAG device, the M-LAG device synchronizes the system MAC address of the VLANIF interface to the peer M-LAG device through the M-LAG synchronization channel.

    12. Run commit

      The configuration is committed.
