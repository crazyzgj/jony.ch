Collecting Error Packet Statistics on Peer-Link Member Interfaces

Context

Packets may be incorrectly changed when being forwarded by a peer-link member interface. It is expected that peer-link member interfaces in an M-LAG can periodically send detection packets to each other to check whether the packets received by the remote end are incorrectly changed. Therefore, a mechanism is proposed to detect packet changes on peer-link member interfaces.

This mechanism is described as follows:
  • When DFS group pairing is successful and peer-link member interfaces are Up, peer-link member interfaces periodically send detection packets.
  • After receiving the detection packets, a remote peer-link member interface checks whether the packets are changed. If the number of changed packets increases, an hwPacketChange alarm is generated.
  • If the number of changed packets does not increase within 5 minutes, the alarm on the interface is cleared.
  • This alarm on a peer-link member interface is also cleared when DFS group pairing fails, the peer-link configuration is deleted, the peer-link member interface goes Down, or the peer-link member interface is removed from the peer-link interface.

Procedure

  1. Run the display fei m-lag synchronization packet-change statistics [ slot slot-id ] command to check statistics about error packet detection packets on peer-link member interfaces of the devices paired into a DFS group on the FEI side.
