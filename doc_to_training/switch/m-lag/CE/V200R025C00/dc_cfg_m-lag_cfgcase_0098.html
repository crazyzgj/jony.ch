Example for Configuring OSPF over M-LAG in V-VBST Mode

Networking Requirements

On the network shown in Figure 1, SwitchA, SwitchB, and SwitchC establish an M-LAG. The M-LAG member interfaces on SwitchB and SwitchC support dynamic routing protocols. A dynamic routing protocol is configured on SwitchA so that it can communicate with the M-LAG through Layer 3 routes.

Figure 1 Network diagram for configuring a dynamic routing protocol for communication with an M-LAG

Configuration Roadmap

The configuration roadmap is as follows:
  1. Configure a routing protocol on SwitchB, SwitchC, and SwitchD to implement Layer 3 connectivity.
  2. Create Eth-Trunk interfaces.

  3. Configure V-VBST.

  4. Configure M-LAG.
    • Configure a DFS group on SwitchB and SwitchC.
    • Configure the link between SwitchB and SwitchC as the peer-link.
    • Bind the user-side Eth-Trunk interface to the DFS group on SwitchB and SwitchC.
  5. Configure an IP address for OSPF over M-LAG on SwitchB and SwitchC. Configure different M-LAG IP addresses on SwitchB and SwitchC. Otherwise, neighbor relationships of the routing protocol cannot be established.
  6. Configure M-LAG member devices to use the specified IP address to establish OSPF neighbor relationships with SwitchA.

Procedure

  1. Configure a routing protocol.

    When OSPF is configured to import direct routes, a routing policy needs to be added to prevent the M-LAG IP address from being advertised.

    # Configure SwitchB. The configurations of SwitchC and SwitchD are similar to the configuration of SwitchB. When configuring OSPF, configure the switches to advertise the 32-bit IP addresses of loopback interfaces.

    <HUAWEI> system-view
[~HUAWEI] sysname SwitchB
[*HUAWEI] commit
[~SwitchB] interface loopback 1
[*SwitchB-LoopBack1] ip address 10.2.2.2 32
[*SwitchB-LoopBack1] quit
[*SwitchB] interface loopback 2
[*SwitchB-LoopBack2] ip address 10.3.3.3 32
[*SwitchB-LoopBack2] quit
[*SwitchB] interface 10ge 1/0/1
[*SwitchB-10GE1/0/1] undo portswitch
[*SwitchB-10GE1/0/1] ip address 192.168.1.1 24
[*SwitchB-10GE1/0/1] quit
[*SwitchB] ospf 1 router-id 11.1.1.1
[*SwitchB-ospf-1] area 0
[*SwitchB-ospf-1-area-0.0.0.0] network 10.2.2.2 0.0.0.0
[*SwitchB-ospf-1-area-0.0.0.0] network 10.3.3.3 0.0.0.0
[*SwitchB-ospf-1-area-0.0.0.0] network 192.168.1.0 0.0.0.255
[*SwitchB-ospf-1-area-0.0.0.0] quit
[*SwitchB-ospf-1] quit
[*SwitchB] commit

    After OSPF is configured successfully, SwitchB, SwitchC, and SwitchD can learn the IP addresses of each other's loopback interface through OSPF and successfully ping each other.

  2. Create Eth-Trunk interfaces and add physical Ethernet interfaces to the Eth-Trunk interfaces.

    The uplink interfaces that connect servers to SwitchB and SwitchC must be added to an Eth-Trunk interface, and the working mode of the Eth-Trunk interface must be the same as that of the Eth-Trunk interfaces on both switches.

    # Create Eth-Trunk interfaces in LACP mode on SwitchB and add member interfaces to the Eth-Trunk interfaces. The configuration of SwitchC is similar to that of SwitchB. The lacp mixed-rate link enable command needs to be configured only when interfaces of different rates are added to an Eth-Trunk interface in LACP mode.

    [~SwitchB] interface eth-trunk 1
[*SwitchB-Eth-Trunk1] mode lacp-static
[*SwitchB-Eth-Trunk1] trunkport 10ge 1/0/4
[*SwitchB-Eth-Trunk1] trunkport 10ge 1/0/4
[*SwitchB-Eth-Trunk1] quit
[*SwitchB] interface eth-trunk 10
[*SwitchB-Eth-Trunk10] mode lacp-static
[*SwitchB-Eth-Trunk10] lacp mixed-rate link enable
[*SwitchB-Eth-Trunk10] trunkport 10ge 1/0/2
[*SwitchB-Eth-Trunk10] quit
[*SwitchB] commit

  3. Set the STP working mode of the switches to VBST.

    # Configure SwitchB.

    [~SwitchB] stp mode vbst
[*SwitchB] commit

    # Configure SwitchC.

    [~SwitchC] stp mode vbst
[*SwitchC] commit

  4. Configure a DFS group on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] dfs-group 1
[*SwitchB-dfs-group-1] source ip 10.3.3.3 peer 10.4.4.4
[*SwitchB-dfs-group-1] dual-active detection enhanced enable
[*SwitchB-dfs-group-1] quit
[*SwitchB] commit

  5. Configure the link between SwitchB and SwitchC as the peer-link.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] interface eth-trunk 1
[~SwitchB-Eth-Trunk1] peer-link 1
[*SwitchB-Eth-Trunk1] quit
[*SwitchB] commit

  6. Bind the user-side Eth-Trunk interface to the DFS group on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] interface eth-trunk 10
[~SwitchB-Eth-Trunk10] dfs-group 1 m-lag 1
[*SwitchB-Eth-Trunk10] quit
[*SwitchB] commit

  7. Configure an IP address for OSPF over M-LAG on SwitchB and SwitchC.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.

    [~SwitchB] vlan 100
[*SwitchB-vlan100] quit
[*SwitchB] interface vlanif 100
[*SwitchB-Vlanif100] ip address 100.100.0.1 255.255.255.0
[*SwitchB-Vlanif100] m-lag ip address 100.100.0.3 255.255.255.0
[*SwitchB-Vlanif100] ospf source sub-address 100.100.0.3
[*SwitchB-Vlanif100] mac-address 0000-5e00-0101
[*SwitchB-Vlanif100] quit
[*SwitchB] commit

  8. Configure M-LAG member devices to use the specified IP address to establish OSPF neighbor relationships with SwitchA.

    # Configure SwitchB. The configuration of SwitchC is similar to that of SwitchB.
    [*SwitchB] ospf
[*SwitchB-ospf-1] area 0
[*SwitchB-ospf-1-area-0.0.0.0] network 100.100.0.0 0.0.0.255
[*SwitchB-ospf-1-area-0.0.0.0] quit
[*SwitchB-ospf-1] quit
[*SwitchB] commit

    # Configure SwitchA.

    [~SwitchA] vlan 100
[*SwitchA-vlan100] quit
[*SwitchA] interface vlanif 100
[*SwitchA-Vlanif100] ip address 100.100.0.2 255.255.255.0
[*SwitchA-Vlanif100] quit
[*SwitchA] commit
[*SwitchA] ospf 1 router-id 11.4.4.4
[*SwitchA-ospf-1] area 0
[*SwitchA-ospf-1-area-0.0.0.0] network 100.100.0.0 0.0.0.255
[*SwitchA-ospf-1-area-0.0.0.0] quit
[*SwitchA-ospf-1] quit
[*SwitchA] commit

Configuration Files

  • SwitchB configuration file

    #
sysname Leaf
#
dfs-group 1
 source ip 10.3.3.3 peer 10.4.4.4
 dual-active detection enhanced enable
#
vlan batch 100
#
stp mode vbst
#
interface Vlanif100
 ip address 100.100.0.1 255.255.255.0
 ospf source sub-address 100.100.0.3
 mac-address 0000-5e00-0101
 m-lag ip address 100.100.0.3 255.255.255.0
#
interface Eth-Trunk1
 mode lacp-static
 peer-link 1
#
interface Eth-Trunk10
 port link-type trunk
 undo port trunk allow-pass vlan 1
 port trunk allow-pass vlan 100
 mode lacp-static
 lacp mixed-rate link enable
 dfs-group 1 m-lag 1
#
interface 10GE1/0/1
 undo portswitch
 ip address 192.168.1.1 255.255.255.0
#
interface 10GE1/0/2
 eth-trunk 10
#
interface 10GE1/0/3
 eth-trunk 1
#
interface 10GE1/0/4
 eth-trunk 1
#
interface LoopBack1
 ip address 10.2.2.2 255.255.255.255
#
interface LoopBack2
 ip address 10.3.3.3 255.255.255.255
#
ospf 1 router-id 11.1.1.1
 area 0.0.0.0
  network 10.2.2.2 0.0.0.0
  network 10.3.3.3 0.0.0.0
  network 100.100.0.0 0.0.0.255
  network 192.168.1.0 0.0.0.255
#
return
