Configuring M-LAG Upgrade in Maintenance Mode

Context

Currently, the following protocols are supported: OSPF, OSPFv3, BGP, BGP4+, and LACP.

Before upgrading an M-LAG member device, perform the following steps in sequence:

  1. Enter the maintenance mode view.
  2. Configure the device to change the cost values of OSPF/OSPFv3 routes or the MED and Local_Pref values of BGP/BGP4+ routes so that traffic sent from other devices to the local device can be switched to another device.
  3. Run an LACP command to switch traffic from the server-side to another device.

After the upgrade is complete, perform the following steps in sequence:

  1. Enter the maintenance mode view.
  2. Run the undo form of the LACP command to switch traffic from the server-side to the local device.
  3. Configure the device to restore the cost of OSPF/OSPFv3 routes or the MED and Local_Pref of BGP/BGP4+ routes to the default values. This enables traffic sent from other devices to the local device to be switched back.

Procedure

  1. Run system-view

    The system view is displayed.

  2. Run maintenance

    The maintenance mode view is displayed.

  3. Perform the following steps depending on the routing protocol configured on the device.

    • OSPF routes

      Run ospf advertise max-cost

      The device is configured to change the costs of all its OSPF LSAs to be advertised to their maximum values.

      After the ospf advertise max-cost command is run, the device changes the costs of all its OSPF LSAs to be advertised to their maximum values. The costs of router-LSAs (Type 1) are increased to 65535, and the costs of summary-LSAs (Type 3 and Type 4), AS-external-LSAs (Type 5), and NSSA-LSAs (Type 7) are all increased to 16711680.

      By default, a device does not change the cost of any of its OSPF LSAs to be advertised to the maximum value.

    • OSPFv3 routes

      Run ospfv3 advertise max-cost

      The device is configured to change the costs of all its OSPFv3 LSAs to be advertised to their maximum values.

      After the ospfv3 advertise max-cost command is run, the device changes the costs of all its OSPFv3 LSAs to be advertised to their maximum values. The costs of router-LSAs (Type 1) are increased to 65535, and the costs of inter-area-prefix-LSAs (Type 3), inter-area-router-LSAs (Type 4), AS-external-LSAs (Type 5), and NSSA-LSAs (Type 7) are all increased to 16711680.

      By default, a device does not change the cost of any of its OSPFv3 LSAs to be advertised to the maximum value.

    • BGP-IPv4 unicast address family and BGP-VPN instance IPv4 address family routes

      Run advertise bgp ipv4-family unicast lowest-priority enable

      BGP is enabled to set the priorities of BGP-IPv4 unicast address family and BGP-VPN instance IPv4 address family routes to be advertised to the lowest.

      The advertise bgp ipv4-family unicast lowest-priority enable and advertise lowest-priority all-address-family peer-up commands can be used to control route advertisement. Either command can enable BGP to set the priorities of routes to be advertised in the preceding address families to the lowest when BGP peers in these address families go from down to up.

      After the advertise bgp ipv4-family unicast lowest-priority enable command is run, the MED and Local_Pref attributes of BGP routes to be advertised cannot be modified through an export routing policy.

      By default, BGP does not set the priorities of BGP-IPv4 unicast address family and BGP-VPN instance IPv4 address family routes to the lowest.

    • BGP-VPNv4 address family routes

      Run advertise bgp ipv4-family vpnv4 lowest-priority enable

      BGP is enabled to set the priorities of BGP-VPNv4 address family routes to be advertised to the lowest.

      The advertise bgp ipv4-family vpnv4 lowest-priority enable and advertise lowest-priority all-address-family peer-up commands can be used to control route advertisement. Either command can enable BGP to set the priorities of routes to be advertised in the preceding address family to the lowest when BGP peers in the address family go from down to up.

      After the advertise bgp ipv4-family unicast lowest-priority enable command is run, the MED and Local_Pref attributes of BGP routes to be advertised cannot be modified through an export routing policy.

      By default, BGP does not set the priorities of BGP-VPNv4 address family routes to be advertised to the lowest.

    • BGP-IPv6 unicast address family and BGP-VPN instance IPv6 address family routes

      Run advertise bgp ipv6-family unicast lowest-priority enable

      BGP is enabled to set the priorities of BGP-IPv6 unicast address family and BGP-VPN instance IPv6 address family routes to be advertised to the lowest.

      The advertise bgp ipv6-family unicast lowest-priority enable and advertise lowest-priority all-address-family peer-up commands can be used to control route advertisement. Either command can enable BGP to set the priorities of routes to be advertised in the preceding address families to the lowest when BGP peers in these address families go from down to up.

      After the advertise bgp ipv4-family unicast lowest-priority enable command is run, the MED and Local_Pref attributes of BGP routes to be advertised cannot be modified through an export routing policy.

      By default, BGP does not set the priorities of BGP-IPv6 unicast address family and BGP-VPN instance IPv6 address family routes to be advertised to the lowest.

    • BGP-VPNv6 address family routes

      Run advertise bgp ipv6-family vpnv6 lowest-priority enable

      BGP is enabled to set the priorities of BGP-VPNv6 address family routes to be advertised to the lowest.

      The advertise bgp ipv6-family vpnv6 lowest-priority enable and advertise lowest-priority all-address-family peer-up commands can be used to control route advertisement. Either command can enable BGP to set the priorities of routes to be advertised in the preceding address family to the lowest when BGP peers in the address family go from down to up.

      After the advertise bgp ipv4-family unicast lowest-priority enable command is run, the MED and Local_Pref attributes of BGP routes to be advertised cannot be modified through an export routing policy.

      By default, BGP does not set the priorities of BGP-VPNv6 address family routes to be advertised to the lowest.

    • BGP-EVPN address family routes

      Run advertise bgp l2vpn-family evpn lowest-priority enable

      BGP is enabled to set the priorities of BGP-EVPN address family routes to be advertised to the lowest.

      The advertise bgp l2vpn-family evpn lowest-priority enable and advertise lowest-priority all-address-family peer-up commands can be used to control route advertisement. Either command can enable BGP to set the priorities of routes to be advertised in the preceding address family to the lowest when BGP peers in the address family go from down to up.

      After the advertise bgp ipv4-family unicast lowest-priority enable command is run, the MED and Local_Pref attributes of BGP routes to be advertised cannot be modified through an export routing policy.

      By default, BGP does not set the priorities of BGP-EVPN address family routes to be advertised to the lowest.

  4. Run lacp force-down

    The device is enabled to forcibly set Eth-Trunk interfaces that are added to an M-LAG to down.

    By default, a device does not forcibly set Eth-Trunk interfaces that are added to an M-LAG to down.

  5. Run commit

    The configuration is committed.

  6. Upgrade the device version. After the upgrade is complete, perform the following steps to switch traffic back to the upgraded device.
  7. Run system-view

    The system view is displayed.

  8. Run maintenance

    The maintenance mode view is displayed.

  9. Run undo lacp force-down

    The device is disabled from forcibly setting Eth-Trunk interfaces that are added to an M-LAG to down.

  10. Perform the following steps depending on the routing protocol configured on the device to implement route traffic switchback.

    • OSPF routes

      Run undo ospf advertise max-cost

      The device is configured to restore the costs of all its OSPF LSAs to be advertised to the default values.

    • OSPFv3 routes

      Run undo ospfv3 advertise max-cost

      The device is configured to restore the costs of all its OSPFv3 LSAs to be advertised to the default values.

    • BGP-IPv4 unicast address family and BGP-VPN instance IPv4 address family routes

      Run undo advertise bgp ipv4-family unicast lowest-priority enable

      BGP is disabled from setting the priorities of BGP-IPv4 unicast address family and BGP-VPN instance IPv4 address family routes to be advertised to the lowest.

    • BGP-VPNv4 address family routes

      Run undo advertise bgp ipv4-family vpnv4 lowest-priority enable

      BGP is disabled from setting the priorities of BGP-VPNv4 address family routes to be advertised to the lowest.

    • BGP-IPv6 unicast address family and BGP-VPN instance IPv6 address family routes

      Run undo advertise bgp ipv6-family unicast lowest-priority enable

      BGP is disabled from setting the priorities of BGP-IPv6 unicast address family and BGP-VPN instance IPv6 address family routes to be advertised to the lowest.

    • BGP-VPNv6 address family routes

      Run undo advertise bgp ipv6-family vpnv6 lowest-priority enable

      BGP is disabled from setting the priorities of BGP-VPNv6 address family routes to be advertised to the lowest.

    • BGP-EVPN address family routes

      Run undo advertise bgp l2vpn-family evpn lowest-priority enable

      BGP is disabled from setting the priorities of BGP-EVPN address family routes to be advertised to the lowest.

  11. Run commit

    The configuration is committed.
