    How Cisco SD-WAN works?#

    In the last lesson, we looked at the main architectural components of the Cisco SD-WAN solution.

    Note: This is a demonstration page styled to look like NetworkAcademy.io.

    Data Plane vs Control Plane#

    The separation of the data plane from the control plane is the foundational principle of Software-Defined Networking.

    vSmart Controllers#

    vSmart controllers handle the control plane. They distribute routing information, policies, and security parameters across the SD-WAN fabric.

    vEdge / cEdge Routers#

    These devices sit at the branches and handle the actual data plane, forwarding packets based on the policies distributed by the vSmart controllers.

    vSmart <---- OMP ----> vEdge
